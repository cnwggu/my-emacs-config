支持编码的摘要
IANA 字符集注册表内的名称:ISO-10646-UCS-4
相关字符集:ISO 10646
描述: 31位编码空间的通用字符集，以 ISO/IEC 10646 标准化为 UCS-4。 它和最新的 Unicode 代码映射表保持同步。
额外的说明: 如果在编码转化程序中使用了这个名字，转换器将尝试识别前缀的 BOM头（字节顺序标记），即表示后续字节的字节序。

额外的说明: 和 UCS-4 相比较，字符串总是设为大端序（big endian）的形式。

IANA 字符集注册表内的名称:UTF-32
相关字符集:Unicode
描述: 32 位单位宽度的 Unicode 转换格式，涉及到了 Unicode 字符集标准。该编码方案和 UCS-4 没有完全相同，因为 Unicode 编码空间限制为 21 位的值。
额外的说明: 如果在编码转化程序中使用了这个名字，转换器将尝试识别前缀的 BOM头（字节顺序标记），即表示后续字节的字节序。

IANA 字符集注册表内的名称:UTF-16
相关字符集:Unicode
描述: 16 位单位宽度的 Unicode 转换格式。值得一记的是，UTF-16 的规格不再和 UCS-2 一样， 因为 Unicode 2.0 引入了代理机制（surrogate mechanism），UTF-16 现在指向了 21 位的编码空间。
额外的说明: 如果在编码转化程序中使用了这个名字，转换器将尝试识别前缀的 BOM头（字节顺序标记），即表示后续字节的字节序。

IANA 字符集注册表内的名称: EUC-JP (首选 MIME 名称) / Extended_UNIX_Code_Packed_Format_for_Japanese / csEUCPkdFmtJapanese
相关字符集: US-ASCII / JIS X0201:1997 (半角假名部分) / JIS X0208:1990 / JIS X0212:1990 的复合
描述: 就像你所看到的，名字来源于扩展 Extended UNIX Code Packed Format for Japanese 的缩写，该编码常用于 UNIX 或类似平台。原始编码方案 Extended UNIX Code 基于 ISO 2022 而设计。
额外的说明: EUC-JP 涉及的字符集和 IBM932 / CP932 是不一样的，后者使用于 OS/2® 和 Microsoft® Windows®。 这些平台上的信息交换，使用 EUCJP-WIN 来代替。
IANA 字符集注册表内的名称:Shift_JIS (首选 MIME 名称) / MS_Kanji / csShift_JIS
相关字符集:JIS X0201:1997 / JIS X0208:1997 的复合
描述: 80 年代初，个人日文文字处理软件进入了市场之时，为了兼容传统编码方案 JIS X 0201:1976，开发了 Shift_JIS。 根据 IANA 对 Shift_JIS 编码集的定义，和 IBM932 / CP932 稍有不同。但 "SJIS"、"Shift_JIS" 的名称常被错误得指向这些编码集。
额外的说明:要使用 CP932，使用 SJIS-WIN 作为替代。
IANA 字符集注册表内的名称:(none)
相关字符集: JIS X0201:1997 / JIS X0208:1997 / IBM extensions / NEC extensions 的复合
